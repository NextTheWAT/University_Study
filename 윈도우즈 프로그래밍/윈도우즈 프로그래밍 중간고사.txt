윈도우즈 프로그래밍 중간고사



1번 - 윈도우 프로그래밍에 대해 서술
그래픽기반으로 명령어를 처리할수있도록 GUI를 사용하는 개념이 윈도우 프로그램과 선점형 멀티태스킹이 특징
GUI : 키보드로 명령어를 직접 입력하지 않고, 마우스로 아이콘이나 메뉴를 선택하여 모든 작업을 수행하는 방식
선점형 멀티태스킹 : 동시에 여러개의 프로그램을 실행하는 멀티태스킹을 하면서
	운영체제가 각 작업의 CPU 이용 시간을 제어하여 응용 프로그램 실행중 문제가 발생하면 해당 프로그램 강제 종료시키고, 	
	모든 시스템 자원을 반환하는 멀티태스킹 운영 방식 


2번 - 객체와 클래스에 대해서 설명하라
객체 : 객체에는 멤버변수, 멤버함수로 구성되어있고, 클래스 타입으로 선언되었을 때를 의미,
	생성될 때 클래스의 모양을 그대로 가지고 생성되고,
	객체들은 상호 별도의 공간에 생성된다.
	
클래스 : 데이터, 액션파트가 있어서 스스로 활동이 가능하다 하지만 접근 한정자가 필요하다
		객체를 만들어내기 위해 정의된 설계도, 틀이다.

3번 코딩




민재 - 
1. 윈도우에 대해서 --- 명령어 입력방식으로 하면 명령어로 하는데 타이핑해야하고 외워야한다.
                         --- 구글, 크롬처럼 아이콘을 클릭하면 넘어가게된다. 이미지가 명령어를 대신 실행시켜주는 것이다.
      --- GUI(그래픽 사용자 인터페이스)이다.  키보드나 명령어를 직접 입력 하지 않고 마우스나 아이콘이나 메뉴를 선택하여 모든 작업을 수행하는 방식
                         --- 그래픽 기반, GUI 개념을 사용하는 윈도우 프로그램이다.
      --- 선점형 멀티태스킹이다. 동시에 여러개의 프로그램을 실행하는 멀티태스킹을 하면서 운영체제가 각 작업의 CPU 이용시간을 제어하여
                              응용 프로그램 실행 중 문제가 발생 하면 해당 프로그램을 강제 종료시키고 모든 시스템 자원을 반환하는 방식을 사용한다.


2. 객체와 클래스의 특징에 대해서 서술하라.
   -- 객체는 클래스타입으로 선언되었을 때를 의미
   -- 객체는 멤버변수와 멤버 함수로 구성되어 있으며 객체들은 상호 별도의 공간에 생성된다. 


 클래스
   -- 데이터파트와 액션파트가 들어가있기에 데이터를 보안하여보안을 위해서 접근한정자가 필요하다! (아는데 까지만 서술하라)
   -- 상속성: 임의의 클래스가 가진 공통된 요소를 다른 클래스에게 물려줄 수 있다.
   -- 다형성: 각기 다른 객체가 "동일한 메시지"를 통해 각기 다른 기능을 수행한다.
   -- 정보은닉: 멤버의 접근 권한을 기본 private으로 두면서, 외부로부터의 접근을 차단한다.
   -- 캡슐화: 관련있는 데이터와 기능을 한번에 묶어서 제공한다.
