데이터베이스
- 데이터베이스가 뭐지?
	- 데이터 집합?
	- 파일?
파일로 관리하면 데이터가 중복이 될수있기 때문에 문제가 될수있다.

어떤 데이터가 필요할까?
성적 관리 시스템 vs 장학금 관리 시스템

성적 관리 시스템
- 학생 데이터
	- 학번, 이름, 학과
	- 총 이수학점, 평점평균
장학금 관리 시스템
- 학생 데이터
	- 학번, 이름, 학과
	- 성적, 소득분위...등등


데이터베이스
- Database : DB
- 한 조직의 여러 응용 시스템이나 사용자들이 @공용할 수 있도록 @통합, @저장된
	운영 데이터의 집합

공용 데이터
- Shared data : 공유 데이터
- 조직의 여러 응용 시스템들이 공동으로 소유하고 유지하며 이용하는 데이터
- 여러 사용자가 서로 다른 목적으로 공용
- 데이터 양 대형화, 구조 복잡

통합 데이터
- Integrated data
- 여러 사용자가 공용할 데이터를 모은다면 중복되는 데이터도 있지 않을까?
- 원칙적으로 데이터 중복(data redundancy)을 허용하지 않음
	- 효율성을 위해 불가피하게 중복을 허용하기도 함
- 데이터베이스는 @통제된 중복(controlled redundancy)이 허용된 통합된 데이터

저장 데이터
- Stored data
- 컴퓨터가 접근할 수 있는 저장 매체에 저장된 데이터

 운영 데이터
- Operational data
- 온갖 종류의 모든 데이터를 모아야 하나?
- 조직 고유의 기능을 수행하기 위해 반드시 유지해야 될 데이터
- 데이터베이스는 단순한 데이터 집합이 아니라 조직의 존재 목적이나
	기능 수행에 반드시 필요한 데이터의 집합
	- 단순 입출력 데이터 X, 임시 데이터 X

@데이터베이스
- 한 조직의 여러 응용 시스템이나 사용자들이 @공용할 수 있도록 @통합, @저장
	된 @운영 데이터의 집합
	=@ 공용, 통합, 저장, 운영

데이터베이스의 특징
- 실시간 접근성(real-time accessibility)
	- 데이터 요구에 대해 실시간 처리 및 응답
- 계속적인 변화(continuous evolution)
	- 현실 세계는 끊임 없이 변하고 있음
	- 현실 세계를 표현하는 데이터베이스 역시 계속적으로 변화해야 함
	- 데이터베이스 상태는 동적임(dynamic)
		- 삽입(insert), 삭제(delete), 수정(갱신: update)
			-> 현재의 정확한 데이터 유지

데이터베이스의 특징
- 동시 공용(concurrent sharing)
	- 여러 사용자가 동시에 원하는 데이터에 접근하여 사용
- @내용에 의한 참조(contents reference)
	- 데이터의 위치나 주소가 아닌 내용(값)에 따라 참조
	- "기말 성적이 90점 이상인 학생을 검색하라."
	= int a 는 a가 가르키는 주솟값에 값을 넣는데
		데이터베이스는 내용에 의한 참조이다

데이터의 유형
- 구조화된 형태에 따른 분류
	- 정형 데이터
	- 반정형 데이터
	- 비정형 데이터
- 데이터의 특성에 따른 부류
	- 수치형 데이터
	- 범주형 데이터
		- 통계적 관점에서 데이터 특성에 따라 적합한 분석 방법을
		  선택하기 위해 데이터 분석 분야에서 주로 활용

구조 : 정형 데이터
- Structured data
- 구조화된 데이터
	- 미리 정해진 구조에 따라 저장된 데이터
- 데이터 구조에 대한 설명과 데이터 내용은 별도로 유지됨
- 예 : 관계 DB의 테이블, 엑셀 스프레드 시트
= 구조가 미리 정의가 되어있기 때문에 값이 무슨 값인지 고민할 필요가 없다

구조 : 비정형 데이터
- Unstructured data
- 정해진 구조가 없이 저장된 데이터
- 예 : 텍스트, 영상, 이미지

구조 : 반정형 데이터
- Semi - structured data
- 구조에 따라 저장된 데이터이지만, 데이터 내용
	안에 구조에 대한 설명이 함께 존재
- 구조를 파악하는 파싱(parsing) 과정이 필요, 보통 파일 형태로 저장
- 예 : HTML, XML, JSON 문서, 웹 로그

특성 : 수치형 데이터
- Numerical data
- 크기 비교와 산술적인 연산이 가능한 숫자 값을 가진 데이터 : 양적 데이터(quantitative) data)
- 이산형 데이터(discrete data)
	- 개수를 셀 수 있는 띄엄띄엄 단절된 숫자 값을 가지는 데이터
	- 예 : 고객 수, 판매량, 합격자 수
	= 정수값의 형태의 데이터 123명 124명 등등 이 사이의 숫자는 없는것.
- 연속형 데이터(continous data)
	- 측정을 통해 얻어지는 연속적으로 이어진 숫자 값을 가지는 데이터
	- 예 : 키, 몸무게, 온도, 점수
	= 키 176, 177 사이의 키가 존재하는 것

특성 : 범주형 데이터
- Categorical data-
- 범주로 구분할 수 있는 값, 즉 종류를 나타내는 값을 가진 데이터
- 크기 비교와 산술적인 연산이 가능하지 않아 질적 데이터(qualitative data)라고도 함
- 명목형 데이터(nominal data)
	- 순서, 즉 서열이 없는 값을 가지는 데이터
	- 예 : 성별, 혈액형, MBTI 유형, 거주 지역(도시)
- 순서형 데이터(ordinal data)
	- 순서, 즉 서열이 있는 값을 가지는 데이터
	- 예 : 학년, 학점 등급, 회원 등급

특성 : 텍스트 데이터
- Text data
- 범주형 데이터도 대부분 문자 데이터지만,
	범주형 데이터는 정해진 (짧은) 텍스트 중 하나
- 가능한 값이 정해져 있지 않은 텍스트 데이터들도 있음
- 예 : 뉴스 본문, 문자메시지, 댓글
- 이외에도 음성, 이미지, 영상 등 다양한 분류 가능
