데이터베이스 9주차

데이터 모델링
• 현실 세계에 존재하는 데이터를 컴퓨터 세계의 데이터베이스로 옮기는 변환 과정
	– 추상화(abstraction)
• 데이터베이스 설계의 핵심 과정


2단계 데이터 모델링
• 개념적 데이터 모델링(conceptual modeling)
	– 현실 세계의 중요 데이터를 추출하여 개념 세계로 옮기는 작업
• 논리적 데이터 모델링(logical modeling)
	– 개념 세계의 데이터를 데이터베이스에 저장하는 구조로 표현하는 작업


데이터 모델
• 데이터 모델링의 결과물을 표현하는 도구
• 개념적 데이터 모델
	– 사람의 머리로 이해할 수 있도록 현실 세계를 개념적으로 모델링하여 데이터베이스의 개념적 구조로 표현하는 도구
▪ 개념 세계로 추상화했을 때 어떤 요소로 이루어지는지 표현
	– 예: 개체-관계 모델(E-R 모델)
• 논리적 데이터 모델
	– 개념적 구조를 논리적 모델링하여 데이터베이스의논리적 구조로 표현하는 도구
▪ 데이터를 어떤 모습으로 저장할 것인지 표현
	– 예: 관계 데이터 모델

• 데이터 구조(data structure)
	– 어떤 요소로 이루어져 있는지, 데이터를 어떤 모습으로 저장할 것인지 표현
	– 표현 대상 정의(개체, 관계)
	– 정적(static)
• 연산(operation)
	– 데이터 구조에 따라 표현된 값들을 처리하는 작업
	– 연산을 통해 한 상태에서 다른 상태로 전이
▪ 데이터베이스 상태: 한 시점에 DB에 표현되어 있는 인스턴스들
	– 동적(dynamic)
• 제약조건(constraint)
	– 데이터 무결성 유지를 위한구조적 측면(허용될 수 있는 개체 인스턴스), 의미적 측면(허용되는 연산의 한계)의 제약사항


개체
• 개체(entity)
	– 단독으로 존재할 수 있으며 다른 것과 구별되는 객체(object)
	– 저장할 만한 가치가 있는 중요 데이터를 가지고 있는사람, 사물, 개념, 사건 등
• 개체 타입(entity type)
	– 이름과 속성(attribute)들로 정의됨
	– 속성: 각 개체만의 고유한 특성이나 상태
• 개체 인스턴스(entity instance)
	– 속성이 실제 값을 가짐으로써 실체화된 개체
• 개체 집합(entity set)
	– 한 개체 타입에 대한 개체 인스턴스들의 집합


개체 – 예
• 개체 타입: 이름과 속성들로 정의됨
• 개체 인스턴스: 속성이 실제 값을 가짐으로써 실체화된 개체
• 개체 집합: 한 개체 타입에 대한 개체 인스턴스들의 집합


속성(Attribute)
• 개체나 관계가 가지고 있는 고유한 특성
• 의미 있는 데이터의 가장 작은 논리적 단위


속성: 단일 값, 다중 값
• 단일 값(single-valued) 속성
	– 특정 개체에 대해 하나의 값을 갖는 속성
	– 예: 학생 개체의 이름 속성
• 다중 값(multi-valued) 속성
	– 한 개체에 대해 여러 개의 값을 가질 수 있는속성
– 예: 학생 개체의 취미 속성, 도서 개체의 저자 속성

속성: 단순, 복합
• 단순(simple) 속성
	– 더 이상 작은 구성원소로 분해할 수 없는 속성
• 복합(composite) 속성
	– 몇 개의 기본적인 단순 속성들로 분해할 수 있는 속성

속성: 유도, 저장
• 유도(derived) 속성
	– 속성의 값이 다른 속성이나 개체가 가지고 있는 값으로부터 유도되어 결정되는 속성
	– 예: ‘과목별성적’ 개체의 ‘평균성적’ 속성
		▪ 등록 개체의 ‘성적’ 속성 값을 계산한 결과
• 저장(stored) 속성
	– 유도 속성 생성에 사용된 속성

널 속성
• 널(null) 값이 허용되는 속성
• 개체 인스턴스가 특정 속성 값을 가지고 있지않다는 것을 명시적으로 표시하기 위함
• 널 값의 의미
	– 해당되지 않는 경우(not applicable)
		▪ 학생 개체 타입의 주민번호 속성: 외국 유학생
	– 알 수 없는 경우(unknown)
		▪ 값이 존재하지만 누락된 경우: 학생 개체 타입의 이름, 키
		▪ 존재 여부를 모르는 경우: 학생 개체 타입의 취미, 휴대폰 번호

키 속성
• 개체 집합에 존재하는 각 개체 인스턴스들을식별하는 데 사용되는 속성
	– 개체 인스턴스들의 키 속성 값은 모두 다름
	– 간단히 키(key)라고도 함
	– 둘 이상의 속성들로 구성하기도 함
• 키 속성으로 적합한 속성이 여러 개면 이 중하나를 키로 사용
• 예:
	– 고객 개체의 고객아이디 속성
	– 도서 객체의 ISBN 속성
	– 고객 개체의 (고객명, 집전화번호)


관계
• 관계(relationship)
	– 개체와 개체가 맺고 있는 의미 있는 연관성
• 관계 인스턴스
	– 실제 속성 값으로 구성된 특정 개체 인스턴스들 간에 맺어진 실제 관계
• 관계 타입
	– 개체 집합들 사이의 대응 관계(매핑)


관계의 유형
• 관계에 참여하는 개체 타입의 수 기준
	– 이항 관계: 개체 타입 2개가 맺는 관계
	– 삼항 관계: 개체 타입 3개가 맺는 관계
	– 순환 관계: 개체 타입 1개가 자기 자신과 맺는 관계
• 매핑 카디널리티 기준
	– 일대일(1:1) 관계
	– 일대다(1:n) 관계
	– 다대다(n:m) 관계


일대일 관계
• 개체 집합 X의 각 원소는 집합 Y의 원소하나와 관련될 수 있고,
• 개체 집합 Y의 각 원소도 집합 X의 원소하나와 관련될 수 있음
• 예: 신랑 – 결혼 – 신부


일대다 관계
• 개체 집합 X의 각 원소는 임의의 수의 Y 원소와 관련될 수 있지만,
• 개체 집합 Y의 각 원소는 많아야 하나의 X 원소와 관련될 수 있음
• 예: 
	– 어머니 – 모자 – 자식
	– 부서 – 소속 – 사원


다대다 관계
• 개체 집합 X의 각 원소는 임의의 수의 개체 집합 Y 원소와 관련될 수 있고,
• 개체 집합 Y의 각 원소도 임의의 수의 X 원소와 관련될 수 있음
• 예: 
	– 학생 – 등록 – 과목
	– 고객 – 구매 – 책


관계의 참여 특성
• 필수적 참여(전체 참여)
	– X-Y 관계에서 개체 집합 X의 모든 개체 인스턴스가 X-Y 관계에 참여해야 하는 경우
	– 예: 교수 – 소속 – 학과
• 선택적 참여(부분 참여)
	– X-Y 관계에서 개체 집합 X의 일부 개체 인스턴스만 X-Y 관계에 참여해도 되는 경우
	– 예: 학생 – 등록 – 과목


관계의 종속성
• 존재 종속(existence dependence)
	– 어떤 개체 y의 존재가 개체 x의 존재에 종속되는 경우
		▪ x: 주 개체(dominant entity); 강한 개체
		▪ y: 종속 개체(subordinate entity); 약한 개체
	– 강한 개체 x를 삭제하면 약한 개체 y도 삭제해야 함
	– 예: 대출 – 대출상환 – 상환
• 강한 개체와 약한 개체는 일반적으로 일대다 관계
• 약한 개체는 강한 개체와의 관계에 필수 참여
• 약한 개체는 강한 개체의 키를 포함하여 키 구성


제약조건
• 매핑 카디널리티, 참여도, 종속성의 성질은 관계 인스턴스들이 유지해야 될 제약조건
• 개체의 삽입, 삭제, 수정 연산 수행 시 이 제약조건 위배 여부 확인 필요


E-R 모델
• Entity-Relationship model
• 1976년 Peter Chen이 제안
• 개체 타입과 관계 타입을 이용해서 현실 세계를 개념적으로 표현하는 방법
• E-R 다이어그램(diagram)
	– E-R 모델을 그래프 방식으로 표현한 것

• 개체 타입
• 관계 타입
• 속성
• 링크
• 레이블: 매핑 카디널리티 표현


E-R 다이어그램 표기법
• 키 속성
	– E-R 다이어그램에서 밑줄로 표시
• 약한(weak) 개체 타입
	– 자기 자신의 속성만으로는 키를 명세할 수 없는 종속 개체 타입
• 강한(strong) 개체 타입
	– 자신의 속성으로 구성된 키를 가진 개체 타입
• 부분키(partial key)
	– 강한 개체가 주어졌을 때, 이와 연관된 약한 개체들을 서로 구별할 수 있는 속성
	– 구별자
• 식별(identifying) 관계 타입
	– 약한 개체를 강한 개체에 연관시켜주는 관계

E-R 다이어그램 표기법
• 필수적(전체) 참여
• 다중 값 속성
• 복합 속성
• 유도 속성
• 관계 타입의 속성


새발 표기법
• 관계 타입
– 식별(identifying) 관계 타입
	▪ 부모 엔티티(주 개체)의 주 식별자가 자식 엔티티
		(종속 개체)의 주 식별자로 사용되는 경우
– 비식별(non-identifying) 관계 타입
	▪ 부모 엔티티(주 개체)의 주 식별자가 자식 엔티티
		(종속 개체)의 주 식별자가 아닌 일반 속성으로 사용되는 경우


논리적 데이터 모델
• E-R 다이어그램으로 표현된 개념적 구조를 데이터베이스에 어떤 형태로 저장할지
	를 표현한 논리적 구조
• 사용자가 생각하는 데이터베이스의 모습 또는 구조
	– 논리적 데이터 모델로 표현된 데이터베이스의 논리적 구조 = 데이터베이스 스키마
		(schema)
• 관계 데이터 모델, 계층 데이터 모델, 네트워크 데이터 모델 등
• relational data model
• 데이터베이스는 릴레이션(테이블)의 집합
• 개체, 관계 모두 릴레이션(테이블)으로 표현


논리적 데이터 모델
• 관계 타입의 표현 방법에 차이가 있음
	– 관계 데이터 모델 – 릴레이션(테이블)@
	– 계층 데이터 모델 – 트리
	– 네트워크 데이터 모델 – 네트워크(그래프)
