1. OSI 계층을 1계층부터 7계층까지 나열하고 각각에 대해 설명하라.

1계층(Physical 계층): 구조화 되지 않은 비트스트림을 물리 매체를 통하여 전송하기 위한
 		     기계적, 기능적, 전기적, 절차적, 스펙을 정의하는 계층이다.
2계층(Data Link 계층): 2계층부터 데이터 구조화된 데이터를 Frame 이라고 부른다 인접
 		       노드간의 신뢰성 있게 전송하기 위한 보류제어, 흐름제어와 같은
 		     전송제어를 담당하는 계층이다.
3계층(Network 계층): 데이터 전송 단위를 Packets이라 부른다 이것은 경우에 따라 조금 더
 		     광범위하게 쓰이지만 여기선 패킷 이라 부른다 가장 중요한 기능은
 		     통신자에서 목적지까지 Network를 통해서 데이터를 보낼 수 있도록
 		     가장 최적의 기능을 찾는 계층이다.
4계층(Transport 계층): 통신을 하는 약 쪽 끝단(End to End 통신)간의 신뢰성 있는
 		    데이터를 전송을 담당하는 계층 (2계층과 동일) 똑같은 전송제어를
 		    한다 하더라도 2계층은 링크단위 이지만 4계층은 양 쪽 끝단의
 		    전송제어를 담당한다
5계층(Session 계층): 응용간 즉 application 간의 대화 혹은 세션을 구성하고 통신 방식을
 		    결정하는 세션 계층이다. 통신방식은 크게 첫 번째 단방향 통신, 두
 		    번째 양방향 통신이 있다. 애플리케이션간의 어떠한 통신 방식을 통해서
		    대화를 할 것인지 결정하는 계층.
6계층(Presentation 계층): 응용간에 대화를 할 때 데이터를 어떻게 표현할 것인가 즉,
 		       데이터 표현에 관련된 기능을 담당하는 계층이다 그래서 표현계층
 		      이라고도 불린다.
7계층(Application 계층): 사용자로 하여금 이러한 OSI 환경에 접속해서 이러한 통신기능들을
 	    	     사용할 수 있도록 사용자에게 응용 인터페이스를 제공하는 계층.


2. 캡슐화(SDU, PCI, PDU, SAP) 과정에 대해 설명하라. 

n과 n+1, n-1 계층으로 설명을해 보겠습니다. 데이터가 하위에있는 계층으로 내려오게 되는데 
자기의 기능을 부여해주기 위해 상위 계층으로부터 받은 데이터에 헤더를 추가하게 된다 
그래서 n 계층의 어떤 기능들은 자기가 붙여놓은 헤더의 필드로서 정의되어 들어간다 
그리고 n-1 계층으로 또 내려가고 자기가 고유의 기능을 헤더로 정의해서 내려받은 데이터에 붙이게 된다 
상위 계층에서 하위계층으로 내려오는 것을 인캡슐레이션 이라고 한다.
n+1 계층의 데이터가 n 계층으로 내려오면 이것은 n 계층의 서비스 데이터 단위 
즉 SDU가 된다 N-SDU 즉 N계층의 PCI 라고 불린다 n 계층 관점에서 보면 n계층의 SDU와 PCI가 합쳐진 
전체 데이터를 n계층의 PDU라고 불린다 그래서 식을 써보자면 N-PDU = N-SDU + N-PCI 이다 n 계층의 PDU 는 n-1 계층의 SDU와 동일하다
링크를 타고 간 데이터는 헤더를 제거하는 과정을 디캡슐레이션이라고 한다.
계층과 계층사이의 전달 인터페이스를 뭐라고 부르냐면 서비스 엑세스포인트 SAP 라고 불린다.
 SAP 식별주소는 TCP/IP 상에서는 포트번호, IEEE 802.2 에서는 LLC SAP번호가 있다. 

3. OSI 계층과 TCP/IP 계층 모델을 비교하라.

OSI모델은 7계층으로 이루어져있고 TCP/IP모델은 5계층으로 이루어져 있다. 4계층까지는 1대1 맵핑관계에 있으며, 
4계층까지 거의 유사하게 동일한 기능을 하고 있다.
TCP/IP모델은 5~7계층을 Application 계층으로 하나로 통합되어서 수용되어지고 있다.
7계층을 5계층으로 줄였을 때 전송 효율에 관해서는 어떠한 것이 더 효율이 높을까.
어ᄄᅠᆫ 모델이 더 많은 헤더를 부가정보로 필요로할까.
부가정보가 많으면 많을수록 더 많은 인캡슐레이션 과정을 거친다 그만큼 헤더도 많아진다 
그래서 전송 효율은 떨어지게 된다 그래서 TCP/IP모델의 계층이 적다는 것은 그만큼 인캡슐레이션 
과정이 적다는 것을 의미한다 대신 Application계층이 OSI 7계층 모델에 비해서 훨씬 더 많은 기능을 필요로 한다. 
왜냐하면 OSI모델의 5,6,7 계층의 모든 기능을 가지고있어야함이다.

4. TCP/IP 계층 모델에서 Bit, Frame, Datagram, Segment, Message 전송 과정을 설명하라.

Link 1에 속해있는 호스트가 송신자 Link 6에 있는 호스트 B가 도착지라 예를 들자
1계층과 2계층까지 동작하는 스위치도있다.
호스트 A에서 라우터 1까지 물리매체를 통해 링크를 타고 비트들이 흐르고, R1에서 R3까지 비트 스트림이 흐른다고 해보자
 R3에서 R4까지 마지막 최종적으로 비트스트림이 R3에서 R4까지 도착을 한다.
동일한 통신을 2계층 관점에서 바라보면 비트가 아니라 프레임이라고 데이터를 칭한다.
라우터를 통해 프레임 데이터가 흐르며 인캡슐레이션, 디캡슐레이션 과정이 1~2계층 사이에서만 일어난다.
OSI 7계층을 보면 전송되는 데이터 단위를 Packets 이라고 불린다. 이 패킷을 네트워크 계층에서 전달할 때 연결형/비연결형 서비스로 나뉜다.
이 차이는 실제 데이터를 전달하기 전에 송수신간의 미리 연결된 데이터를 보내느냐 마느냐 차이인데, 패킷을 전달함에 있어서 
비연결형으로 보내는 것을 Datagram 이라고 부른다.
인캡슐레이션과 디캡슐레이션 역시 1~3계층까지 이루어진다는 것도 알 수 있다.
OSI계층에서 물리계층은 구조화 되지 않은 비트스트림의 전송이라 했고 2계층 부터는 구조화 하여 데이터를 
Frame, 3계층에서는 Packet 혹은 Datagram 이라고 부른다.
그리고 2계층에서 사용하는 주소는 MAC 주소 또는 물리 주소라 부르고 3계층에서 사용하는 주소를 논리주소라 부르며 
TCP/IP에서의 3계층에 사용되는 주소는 IP주소라고 한다.
그리고 전송 계층 통신에 사용되는 단위를 Segment라고 부르며 여기서의 차이점은 4계층부터 1계층 사이의 인캡슐레이션이 
이루어지며 디캡슐레이션은 역시 3계층까지밖에 이루어지지 않는다.
동일한 환경을 봤을 때 5계층부터 인캡슐레이션이 이루어진다면 이후에 디캡슐레이션, 인캡슐레이션 반복은 역시 3계층까지/부터 이루어진다. 
최종 목적지도 동일하게 먼저 바라봤던 5계층까지 디캡슐레이션이 이루어진다. 데이터 단위는 1계층에서는 비트, 2계층에서는 Frame, 3계층에서는 
Packet or Datagram 이며 4계층은 Segment라고 한다면 마지막 5계층에서 보내는 데이터 단위는 Message라고 칭한다.


5. MAC 주소, IP주소, 포트번호를 비교 설명하라.

MAC address : 물리주소(Physical address), L2 주소이다.
		링크 상에서 NIC를 구별하며 48Bits 이다 이것은 OUI cod(24bits) + Serial(24bit) 으로 나뉜다.
IP address : 논리주소(Logical address), L3 주소이다.
	     인터넷상에서 호스트를 구별하며
	     IPv4 address (32bits)와 IPv6 address가 있다 IPv4 는 Network ID(Prefix) + Host ID(Suffix) 로 나뉜다 IPv6는 아직까지는 쓰여지지않고 있다.
Port Number : L4 주소이다.
	       Transport 계층에서 상위 응용 process를 구별한다.
	       16bits 이며
	       Well-known Port number :1023번 이하 포트번호 – 인터넷 표준기구에서 웰노운 포트번호를 공식적으로 고정적 포트번호를 할당해두었다.
 - FTP(20,21), SSH(22), Telnet(23), SMTP(25), POP3(110), HTTP(80), HTTPS(443)
만약 동일한 IP를 사용한다면 물리적으로 불가능함. IP주소 충돌이 일어난다는 메시지를 받는다 MAC 주소는 링크상에서 NIC를 구분하기위한 주소이기 때문에 동일한 주소를 사용할 수 없다. 
논리적으로는 동일한 MAC 주소를 가질수있을 수 없다 하지만 있을수도있다.
