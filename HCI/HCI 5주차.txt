HCI 5주차


User(사용자)

사람(human) : 보편적 특성, 복잡함 회피, 모양보다 색상차이, 시청각적 단서를 활용한 정보 탐색
사회인 : 유행, 사회 경제적 상황, 기술 트렌드, 세대적 성향
서비스 사용자(좁은 의미) : UX 디자인을 ㅜ이해 persona 활용한 구분, 또는 targeting


사용자 3가지 관점 : 사용자, 사회인, 사람 => p.23
설문조사로 알 수 있는 사용자 특성 => p.24
사용자의 데이터 분석으로 알 수 있는 사용자 특성 => p.24


설문조사를 통해 알 수 있는 사용자 특성 
이용형태 - 현재 해당 서비스(군)을 어떻게 사용하고 있는지
		- 예) 주로 사용하는 서비스, 어떤 경로를 통해 유입되는지
맥락 - 사용장소, 시간, 상황 등에 대해 파악
의견, 만족도 - 해당 서비스(군)의 만족도 파악
기술적 태도 - 신기술에 대한 수용도, 태도

사용 통계 분석을 통해 알 수 있는 사용자 특성
유입/유출 경로 - 어느 채널, 터치포인트를 통해 들어오고 나갔는지
이동경로 - 순차적인 이용흐름, 맥락에 따른 이용흐름, 특정 화면까지의 이용흐름
관심사 - 특정 페이지에 머문 시간, 특정 페이지에서 선택한 메뉴/기능
탐색행위 - 자주 이용한 검색어, 필터옵션, 해시태그 


경험 vs 사용자 경험
경험 -> 사용자 경험
고객, 소비자, 구매자 -> 사용자 => p.25, p.24
	- 특정 서비스와 관련된 경험만을 주목


사용자 경험(UX) - 예를 들어 정의해 보자
- 사용자가 디지털 제품이나 서비스 사용에서 갖게 되는 모든 지식과 기억과 감정
- 사용자가 디지털 제품이나 서비스를 사용하기 전, 중, 후에 일어나는
	감정, 신념, 선호도, 지각, 신체적 정신적 반응이나 행동을 포함하는 매우 넓은 개념
- 특정 서비스를 이용할 때 사용자들이 보이는 생각, 감정 행동
- 은행 : 디지털 뱅킹을 통한 은행 업무 처리를 위한 시스템 사용
- 통신 : 통신 상품 사용, 미디어 사용 등 광범위
- 쇼핑 : 접근, 유입, 탐색, 조회, 구매, 이벤트, 관리활동, 프로모션, 이탈
- SNS : 가입, 조회, 등록, 관리, 커뮤니케이션, 카메라/미디어, 서비스 연동 등


경험요소
실제 사용자 경험과 연관된 것. (경험요소 <-> 비경험요소)
사용자가 관심 갖는 경험요소와 불필요하게 여기는 것 구분
경험 요소들의 우선순위 결정

사용자의 니즈 + 창의적인 아이디어

경험요소 : 좋은 구슬(What)을 잘 꿰어야(How)
	What : 기능, 정보, 외형 UI, 디자인 등
	How : 이용절차, 배열, 안애, 인증, 에러처리, 커뮤니케이션 등

미래 기대
- 과거의 경험에 기반, 현재의 사용,
	사용 이후의 결과와 피드백에 대한 기대
	적절한 UX가 제공되지 않으면 대안이나 차선책 탐색

시간 흐름에 따른 정보 니즈
1. 과거에 일어난 일의 정보 : 특정 시점의 정보
2. 과거 - 현재까지의 변화 : 과거 - 특정시점 까지의 누적된 정보
3. 현재의 정보
4. 앞으로의 변화 패턴 : 정보 추이, 정보 변화의 패턴, 경향
5. 미래 어느 시점에서의 정보 예측


분류, 그룹핑
- 사용자의 니즈, 원하는 정보의 성격이 다양하게 존재
1. 어느 것이든 상관없이 흥미 있는 것들을 보고 싶다.
2. 원하는 정보를 바로 찾고 싶다
3. 특정 영역에 해당하는 정보를 모두 보고 싶다
4. 목적을 달성하는데 적합한 정보 고르기
5. 목적을 달성하는데 필요한 부분을 채워 나가기 


목적
- 목적 의식의 강도 다양하게 존재
1. 약한 목적의식 : 심심해서, 기분 전환
2. 습관적인 확인, 굳이 지금 할 필요 없는 활동
3. 내적동기가 있으나 생략해도 무방함
4. 지금 해야 하는 주기가 짧은 활동들
5. 반드시 지금 해야 하는 주기가 긴 활동들


좋은 UX

사용자들이 느끼는 보편적인 고충, 갈등, 어려움이 없는 것
실제 서비스를 이용하는 구체적인 맥락 참작
정확한 결과가 명확하고 세심하게 전달
주어진 콘텐츠의 양, 이용흐름의 복잡성, 필요한 정보/데이터에 따라서 사용자에게
	필요한 도구와 자원을 적절하게 제공
그 서비스만의 정체성이 분명한 것
대상 사용자의 특성을 감안하여 좋은 기본값을 제공
기능이나 이용흐름 제시되는 정보의 양이 대상 사용자가 수용 가능
