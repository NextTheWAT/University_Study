인덱스 컬러모드
컬러 번역기
• 프레임 버퍼 내용은 CLUT (Color Lookup Table)의 인덱스
• 실제 컬러는 CLUT의 우측 칼럼에 의해 결정

index칼라 는 8비트씩 풀칼라 이다 R,G,Bd (24비트)
index 0번부터 256가지의 색상이 있다 여러가지 색


디지털 하프 토우닝
점의 크기 대신 개수를 조절
• 우리 눈이 4개의 화소를 한 단위 평균적 밝기를 인식
• 실질적 해상도는 감소(화면에 표현되지 않는 경우)

CMYK도 잉크를 절약하는 기법이다


디더링
해상도 감소가 없음 :
- 색상의 표현이 불가능할 때 다른 색상들을 섞어서
비슷한 색상을 내기 위해 색상의 개수를 줄이는 기술
(예: 도트프린터, 신문 등)
• 화소에서 화소로의 사상 -> 해상도 유지
• Ex. 오류 확산(Error Diffusion Dither) 

• 예: 중앙화소 회색도: 200, 
흑백 모니터인 경우(0 또는 255)인 경우, 
255로 표현-> 오류 -55, 
중앙화소의 오른쪽 화소 계산: 원래 화소값에
-55*7/16을 더하여 가까운 값으로 결정
	200이라면 ex) 200 + -55*7/16 해서 나온 값에서 가까운 값으로 간다

• Ex. 패턴 디더(Patterned Dither)
 모든 화소의 회색도를 최대 8로 정규화(화소값* 8/255)
	ex) 127 이라면 127*255/255 = 4
	ex) 255 라면 255*255/255 = 8
 정규화 값이 행렬 값보다 크면 화소에 가장 가까운 색으로, 작으면 배경색으로
 예: 정규화 결과가 1.1인 경우 3행2열에 해당하는 경우 1.1<2 이므로 배경색

그래픽 분야 표준의 목표
주전산기 독립성(Host Machine Independence) 
• 동일한 프로그램을 가지고서 다양한 모든 하드웨어에서 사용할 수 있어야
한다. 
장비 독립성(Device Independence) 
• 동일 기능을 수행하는 입출력 장비의 종류가 달라도 프로그램 명령은 동
일해야 한다. 
프로그램 언어 독립성(Programming Language Independence) 
• 프로그램 작성에 어떠한 프로그램 언어를 사용해도 된다. 
운영자 이식성(Operator Portability) 
• 새로운 프로그램 사용법을 누구라도 쉽게 터득할 수 있어야 한다.

Q1) 원영상 f(x,y)가 8비트로 회색도를 표현할 때, 패턴 디더링을 수행하고자 한다. 회색도를 최대 15로 할 때, 정규화 공식을 쓰시오. 그리고 51의 회색도를 정규화 하시오.

패턴 디더링을 수행할 때, 회색도의 최대값이 15가 되도록 정규화 공식은 다음과 같습니다.
normalized_value = floor((original_value / 255) * 15 + 0.5)
여기서 original_value는 0부터 255까지의 값 중 하나이며, normalized_value는 0부터 15까지의 값 중 하나입니다.
또한, 51의 회색도를 정규화하면 다음과 같이 계산됩니다.
normalized_value = floor((51 / 255) * 15 + 0.5) = 3
따라서, 51의 회색도는 정규화한 결과 3이 됩니다.
