Class Bean{
private String name;   //프라이베잇 설정
				// 값을 바꾸려면 세터를 사용해야함
}				// 같이 따라오는 게터도 있음

public void SetName( ? ){	//값을 설정
this.name = name;			//매개변수 name값을 넘겨받아 자신의 변수인 this.name에 저장
}
public void GetName( ? ){	//값을 접근
return name;				//name 변수에 저장된 값을 반환해 주도록 한다.
}

 JSP 페이지에 HTML을 제외한 다른 많은 JSP 코드들이 있으므로 해서
    JSP 페이지를 이해하기 어려워짐.
 자바빈즈는 JSP에 안의 수많은 자바 코드들이 담당했던 일들을 독립적
    으로 처리하기 위한 부품과 같음.
 자바빈즈와 같은 컴포넌트 기술을 이용하여 작성할 수 있게 하여 효율성, 
    재사용성 등의 장점을 가짐.

자바빈즈 만들기
1. 정보를 저장하는 변수는 모드 private로 선언
2. private로 선언된 변수의 값을 저장하는 메소드를 만듬
	메소드의 이름은 setxxx() 형식으로 만든다
	Xxx는 변수의 이름과 같으며 첫 글자는 대문자
3. private로 선언된 변수의 값을 가져오는 메소드를 만듬
	메소드의 이름은 getxxx() 형식으로 만든다
	Xxx는 변수의 이름과 같으며 첫 글자는 대문자
4. setXxx() 메소드와 getXxx() 메소드를 public으로 선언


setter - 가져오는
getter - 설정해주는
 setXxx() 메소드를 setter라고 부릅니다.
 getXxx() 메소드를 getter라고 부릅니다.
이 메소드들을 사용하여 값을 간접적으로 접근

빈 관련 태그
<jsp:useBean id="변수명" class="클래스명" scope="..."/>
<jsp:setProperty name="..." property="..." value="..."/>
<jsp:getProperty name="..." property="..."/>

useBean을 사용
ex) id, pw, name이 있다면 3개의 get,set이 있는 인스턴스 할당


<jsp:useBean id="myBean" class="09.SimpleBean" scope="page"/>
myBean이 주소값을 가지고 있는데 그 주소값이 인스턴스를 가르킨다 myBean 이라는
ex) a myBean = new a(); 와 비슷함
class - 패키지.클래스명
scope="page" - 페이지 단위로


scope의 종류
page 	- 해당 JSP 페이지 내에서만 존재합니다.
request 	- 사용자의 요청(request)을 처리하는 동안 존재합니다.
session 	- 사용자가 최초에 접속하여 세션이 종료되기까지 존재합니다.
application - 해당 어플리케이션이 살아 있는 동안 함께 존재합니다.
