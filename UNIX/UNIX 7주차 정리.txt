셸의 종류와 변경
- 셸 ?
   - 사용자와 커널 사이의 중간 역할
   - 사용자가 입력한 명령을 처리하고 실행 결과를 알려줌
   - 편리한 사용을 위해 다양한 기능 제공
- 셸의 종류
   - 본 셸(sh), 콘 셸(ksh), C 셸(csh), 배시 셸(bash) 등
   - 종류 확인하기
     - 프롬프트로 확인 : C 셸은 % / 본 셸, 콘 셸, 배시 셸은 $ 
     - 명령으로 확인
- 로그인 셸과 서브셸 ?
   - 로그인 셸 : 사용자가 로그인 직후 자동 생성되는 셸
   - 서브셸 : 사용자가 직접 실행한 셸

문자열의 출력
- echo
   - 셸 명령과 유틸리티 모두 제공
   - 지정한 문자열 출력
- print

셸 특수 문자
   - 셸 명령
   - 옵션이 제공되어 echo보다 편리

- * 특수 문자
   - 가장 자주 사용되는 특수 문자 → 임의의 문자열을 나타냄

- ?와 [ ] 특수 문자
   - 모두 하나의 문자를 나타내는 데 사용
   - ?는 임의의 한 문자, [ ] 괄호 안에 포함된 문자 중 하나
   - [ ] 다른 특수 문자와도 혼합해 사용할 수도 있다.

- ~와 – 특수 문자
   - 디렉토리를 나타내는 특수 문자
   - ~가 단독 사용 → 사용자의 홈 디렉토리
   - -는 cd 명령으로 디렉토리를 이전하기 직전의 작업 디렉토리

- ;과 ｜특수 문자
   - 명령과 명령을 연결
   - ;은 연결된 명령을 왼쪽부터 차례로 실행
   - ｜는 왼쪽 명령의 실행 결과를 오른쪽 명령의 입력으로 전달 @
   - ｜는 여러 명령을 연결해 사용 가능
| - 파이프는 왼쪽에 있는 입력값을 오른쪽 출력값으로 전달

- ‘ ‘와 “ “ 특수 문자
   - 문자열 안에 사용된 특수 문자의 의미를 없애는 기능
   - 작은 따옴표는 모든 특수 문자를 처리
   - 큰 따옴표는 $, `, \를 제외한 모든 특수 문자들을 일반 문자로 처리

- ` 특수 문자
   - 명령 실행 결과를 문자열로 받고 싶을 때 사용
   - 작은 따옴표와 혼동 주의

- ＼(\) 특수 문자    중요x
   - ＼ (역 슬래시)는 특수 문자를 일반 문자처럼 사용
   - 특수 문자 앞에 사용되며 특수 문자의 효과를 제거
   - 키보드에서 ＼대신 \를 사용

리디렉션(re-direction) @
- >, >>, < 특수 문자 
   - 입출력 방향을 바꾸는 특수 문자


- 표준 입출력 장치
   - 파일 디스크립터
   - 입출력 장치를 변경할 때 사용

유닉스는 장치도 파일이다@

출력 리다이렉션 : >, >>
    파일 덮어 쓰기 : 명령  >  파일명
    파일에 내용 추가하기 : 명령 >> 파일명
- 파일 덮어쓰기 :>
   - 새로운 파일로 생성, 기존 파일의 내용은 없어짐
- 파일에 내용 추가하기 :>>
   - 기존 파일의 끝에 내용 추가

입력 리다이렉션 : <
    명령 < 파일명
- 표준 입력을 키보드에서 파일로 변경
