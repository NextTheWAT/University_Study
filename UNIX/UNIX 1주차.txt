시험에 나옴
1. 운영체제의 세가지 주요 목적은 무엇인가?
  1. 하드웨어 자원 관리
  2. 프로그램 실행
  3. 사용자 인터페이스 제공

2. 다중 프로그래밍의 주요 장점은 무엇인가?
 - 메모리의 사용자영역에 프로그램이 여러개 탑재되어 있고,
   CPU가 이를 번갈아가며 수행한다. (더 알아보자)
 - 프로그램 관리도 편하고 리소스 관리도 편함

3. 운영체제의 역할은 무엇인가?
  1. 프로세스 관리
  2. 기억장치 관리
  3. 입출력장치 관리
  4. 자원 관리

4. 다중 프로그래밍과 다중처리시스템의 차이를 설명하시오
 - 다중 프로그래밍은 cpu가 이를 번갈아가며 수행하고
 - 다중처리 시스템은 cpu를 이용하여 병렬로 처리하는것을
   가리킨다

5. 운영체제의 기능과 서비스를 구별하시오.
 - 운영체제의 기능 3. 에 나와있음
 - 운영체제의 서비스는 사용자에게 편리성과 효율성을 제공한다
   
6. 운영체제의 목적으로 가장 거리가 먼 것은?
 - 가 사용자 인터페이스 제공
 - 나 주변 장치 관리
 - 다 데이터 압축 및 복원 =
 - 라 신뢰성 향상

7. 한정된 시간 제약 조건에서 데이터를 분석하여 처리하는
    시스템으로 비행기 제어 시스템이나 교통 제어 등에
    사용되는 운영체제의 종류는?
 - 가 분산 처리 시스템
 - 나 일괄 처리 시스템
 - 다 실시간 시스템 = 
 - 라 병렬 처리 시스템

8. 운영체제의 기능으로 적당하지 않은 것은?
 - 가 컴퓨터 시스템의 초기화 기능
 - 나 효율적인 자원 관리와 할당 기능
 - 다 고급 언어로 작성된 프로그램을 기계어로 번역하는 기능 = (운영체제가 아니라 컴파일러 자바, 이클립스 비쥬얼스튜디오 등)
 - 라 오류 검사 및 복구 가능

유닉스의 특징            시험에 나옴 1개당 1점
1. 대화형 시스템
2. 다중 사용자 시스템
3. 다중 작업용 시스템
4. 높은 이식성, 확장성, 개방성  (높은 이개확)
5. 계층적 파일 시스템 = 
6. 부가적 기능 제공 다양

유닉스 시스템의 구조
커널
 - 컴퓨터 자원 관리 (핵)
    - 메모리, 파일 시스템, 장치관리

 - 쉘 (명령 입,출력)
    - 사용자 인터페이스 제공
    - 사용자의 명령을 입력받아 처리한 후 결과를 출력

커널, 쉘이 유닉스에 가까움

 - 유틸리티 (데이터 압축복원, 등등)
    - 각종 프로그래밍 개발 도구, 문서 편집도구 등 (카톡 등)
유틸리티가 직접 하드웨어를 접속할수없음 왜냐하면 이식성도 깨지고
확장성도 깨짐
쉘 - 커널 - 하드웨어로 거쳐가야함
